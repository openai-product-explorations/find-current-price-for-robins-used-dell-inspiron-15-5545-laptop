Figuring out what's in here

I did a buncha work here but I don't remember all what I did or what was left to do.

2025-06-01 21:22 (UTC-7) by Daniel

I did a .fork of https://github.com/pflagerd/python-venv-bootstrap

Note that .fork refers to a manual forking process involving hidden files .fork and .forks in this repository and its parent's repository respectively.

I also did a clone of https://github.com/pflagerd/blog-bootstrap and renamed it blog/

This was added to https://github.com/pflagerd/youtube-to-text via the .gitrepo mechanism, and as far as https://github.com/pflagerd/blog-bootstrap is concerned is a branch called youtube-to-text which was added at the branch youtube-to-text-begin.

.gitrepos doesn't currently record the sha1 of https://github.com/pflagerd/youtube-to-text when the other repository was added, but you can deduce it from the revision of the .gitrepos and the addition of the directive to ignore blog/ in .gitignore.

I deduced that extract-audio-from-video.py and download-from-youtube.py were added as projects to experiment with these two operations prior to merging them into something else.

So I .venv/bin/pip installed their python package dependencies to the .venv created by SETMEUP, namely ffmpeg and yt-dlp.

I'm currently working on creating a transcript from Government Report Reveals A LOT of People Are About to Lose Their Jobs.

So firstly, I want to download that video from https://www.youtube.com/watch?v=qtxkZcpnxcA

So I created a launcher called download-from-youtube which suceeded thus:

which resulted in creation of the following file:

I opened that file in Haruna and it played just fine.

I then created another launcher extract-audio-from-video, passed it the name of the mp4, and it ran successfully thus:

which produced the following file:

-rw-rw-r-- 1 oy753c oy753c 19M Jun  1 22:04 qtxkZcpnxcA.mp3

Opened this file in vlc player and it worked just fine. the default player associated with .mp3s did not want to play this file, but played another one. Weird.

Next I created another launcher called youtube-audio-to-text, which failed due to a missing API_KEY environment variable.

Got an api key from openai and created the environment variable and voila:

-rw-rw-r-- 1 oy753c oy753c 22K Jun 1 22:18 qtxkZcpnxcA.mp3.txt

And after about 30 seconds:

-rw-rw-r-- 1 oy753c oy753c  22K Jun  1 22:18  qtxkZcpnxcA.mp3.txt

Incredible.
